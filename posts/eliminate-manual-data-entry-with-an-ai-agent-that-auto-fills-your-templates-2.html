Eliminate Manual Data Entry with an AI Agent That Auto-Fills Your Templates

December 28, 2025 • By Yi Qu

When I was working in the lab, I saw the same scene again and again. Someone would open a raw data PDF file (scanned or searchable), lean closer to the screen, zoom in, zoom out, and copy numbers into Excel one cell at a time. Then they would pause, squint at the units, fix a tiny typo, and start over. After a few minutes, it was clear this wasn’t a one-time annoyance. It was a daily routine.

This is not “science.” This is data entry.

So here’s the idea: build an AI agent that acts like your data entry buddy. You upload a PDF → it reads it → it pulls out the values you care about → and it fills your Excel template while leaving formulas untouched.

What you are building

Your agent follows one simple workflow. It takes in a PDF, extracts embedded text if the file is searchable, and uses OCR only when the PDF (or the relevant section) is image-based or the extracted text is missing or garbled. Then it identifies the key lines, maps each value using your Field Mapping Guide, and exports everything into your Output Template with flags for anything uncertain.

Why the setup matters

Raw data file layouts might change. A label moves, a header is renamed, or a software update adds a new section. If the agent has to guess, it will eventually guess wrong.

That is why the most reliable approach is a two-stage workflow. First, you teach the agent your format using setup files (discussed in the following sections). Then, you upload the real raw data PDFs, and the agent fills the template using the same rules.

Real example (formulas must stay)

In this example, your output file is an Excel template that already contains formulas such as Average and Standard Deviation. The agent must fill only the input cells and must not overwrite any formula cells. Once the numbers are filled, Excel calculates the summary automatically.

If you want the exact files used in this example, download Appendix A (Field Mapping Guide) and Appendix B (Output Template) at the end.

Step-by-step: build and use the agent

Step 1 — Prepare the Field Mapping Guide

Create a short Word document that explains what each label means and how values should map into your template. This guide is what prevents the agent from guessing when a report contains extra numbers.

If you want a ready-to-use example, download Appendix A.

Step 2 — Prepare the Excel Output Template

Create an Excel template with the exact columns you want. Put formulas wherever you want Excel to calculate results automatically. Make it clear which cells are inputs and which cells are formulas, because the agent must only write to inputs.

If you want a ready-to-use example, download Appendix B.

Step 3 — Add the System Prompt in agent Instructions

Put the System Prompt in the agent’s permanent Instructions area. This is the job description that should not change: extract text first, OCR only if needed, map fields using the guide, and never overwrite formula cells.

If you want a copy/paste-ready System Prompt, download Appendix C.

Step 4 — Use the prompts (even if there’s only one user prompt box)

Most tools give you one permanent place for the System Prompt, and only one chat box for user messages. That’s fine.

You treat the Setup request and the Run request as two reusable scripts that you paste into the same chat box at different times. The first time you use the agent for a new report style, you paste the Setup request and upload the three setup files together: (1) a sample raw report PDF, (2) Output_Template.xlsx, and (3) Field_Mapping_Guide.docx. After the agent confirms what it learned, you paste the Run request and upload the real raw PDFs. On future days, you usually skip Setup and use only the Run request.

To make this easy, you can download the exact Setup and Run prompts in Appendix D and Appendix E.

If an AI can’t give you a downloadable Excel output file, the workaround is simple: request CSV outputs or column-aligned tables and paste them into a formula-protected template. This keeps calculations intact while avoiding file-transfer limitations.

Step 5 — QA loop (how you make it reliable)

Your first run will not be perfect, and that’s normal.

When you spot a mistake, don’t fight the model. Improve the guide. Add one clearer mapping rule and one worked example to the Field Mapping Guide so the agent stops guessing. Over a handful of real PDFs, the workflow becomes consistent for that report style.

If you want something to practice with before using your own reports, I’m also providing sample raw data PDFs (Appendix F) and a small QA test set (Appendix G) so you can run the full workflow end-to-end.

Ending note: this works across different AI tools

This pattern is platform-agnostic. Whether you build your agent in ChatGPT, Copilot, or another AI system, the reliable combo is the same: clear Instructions, a mapping guide, an output template, and a simple QA loop.
