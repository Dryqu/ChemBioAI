Build a Practical Make.com Automation: From Google Forms to AI Decisions

December 25, 2025 • By Yi Qu

A few weeks ago, I attended an event at Make’s Raleigh, North Carolina office. It’s a really nice office, the employees were welcoming, and the presenter walked us through a live demo where we built a scenario together. I found the workflow practical and easy to adapt, so I’m sharing a version of that scenario here.

If you’re also considering Make for building automation, this can serve as a solid starting point you can copy, run, and customize.

What You’ll Build (High-Level Flow)

Google Form submission → Apps Script → Make Webhook →

Route A: Gmail “thank you” email to respondent (only if email exists)

Route B: AI analysis

Final: Gmail email to you with the AI result + raw answers

Step 1) Create Your Google Form Survey

Create a Google Form called something like: “Service Feedback Survey”.

Strong recommendation (for reliable emails)

In Form Settings:

  • Turn ON Collect email addresses

This allows Apps Script to read the respondent email automatically.

If you can’t use “Collect email addresses,” add a required question:

  • “Your email” (Short answer)

Step 2) Add Apps Script to Your Google Form

Open the Google Form:

  • Extensions → Apps Script

Paste the code below

Replace the webhook URL with your Make webhook URL (you’ll create it next)

✅ Apps Script (copy/paste)

function onFormSubmit(e) {
  var response = e.response;
  var itemResponses = response.getItemResponses();

  // Build a clean Q/A dictionary
  var answers = {};
  var emailFromQuestion = "";

  itemResponses.forEach(function (ir) {
    var q = ir.getItem().getTitle();
    var a = ir.getResponse();
    answers[q] = a;

    // If you used a question like "Your email"
    if (q.toLowerCase().includes("your email")) emailFromQuestion = a;
  });

  // If form collects emails, this will work
  var respondentEmail = "";
  try {
    respondentEmail = response.getRespondentEmail() || "";
  } catch (err) {
    respondentEmail = "";
  }

  // Prefer collected email; fallback to typed email question
  var surveyRespondentEmail = respondentEmail || emailFromQuestion || "";

  var payloadObj = {
    respondentEmail: surveyRespondentEmail,
    timestamp: (response.getTimestamp && response.getTimestamp())
      ? response.getTimestamp().toISOString()
      : new Date().toISOString(),
    responseId: (response.getId && response.getId()) ? response.getId() : "",
    answers: answers
  };

  // ✅ Replace this with your Make webhook URL
  var url = "https://hook.us2.make.com/REPLACE_WITH_YOUR_WEBHOOK_ID";

  UrlFetchApp.fetch(url, {
    method: "post",
    contentType: "application/json",
    payload: JSON.stringify(payloadObj),
    muteHttpExceptions: true
  });
}

Step 3) Create the Trigger (Required)

In Apps Script:

  • Click Triggers (clock icon)
  • Add Trigger

Settings:

  • Function: onFormSubmit
  • Event source: From form
  • Event type: On form submit

Save and authorize permissions

You may see: “Google hasn’t verified this app”

That’s normal for personal Apps Script projects. Click:

  • Advanced → Go to (unsafe) → Allow

Step 4) Build the Make.com Scenario

Module 1 — Webhooks → Custom Webhook

  • Create a new scenario
  • Add Webhooks → Custom webhook
  • Click Add, name it: survey_response_received
  • Copy the webhook URL and paste it into your Apps Script code

Now capture a sample:

  • Click Run once in Make
  • Submit a test form response

Make will “learn” your payload fields

You should see:

  • respondentEmail
  • timestamp
  • responseId
  • answers

Step 5) Add a Router (Two Paths)

Add a Router after the webhook module.

Route A — Send “Thank You” Email to Respondent

Add a filter so this only runs if there is an email address.

Filter condition

  • Left: respondentEmail
  • Operator: Is not empty
  • (no right value)

Then add module:

  • Gmail → Send an email

To: {{respondentEmail}}

Subject: Thanks — we received your survey response

Body:

Hi,

Thank you for completing our survey.
We received your response on {{timestamp}}.

We appreciate your time!

— Team

Step 6) AI Analysis (Route B)

On Route B, add:

  • Make AI Agents → Run an agent

AI Input (map from webhook)

Use the original answers verbatim:

Survey submission (verbatim answers):
{{toJSON(answers)}}

AI Instructions (copy/paste)

You are an assistant analyzing a survey response.

Use ONLY the provided answers. Do not invent facts.

Return JSON only with:
- recommendation: "Positive" | "Neutral" | "Needs Follow-up" | "High Priority"
- confidence: 0-100
- summary: 3-6 bullets
- justification: 5-10 sentences citing specific answers
- follow_up_questions: array

Step 7) Email You the Result + Verbatim Answers

Add module after AI:

  • Gmail → Send an email

To: (your email)

Example: John.Smither@gmail.com

Subject:

Survey AI Result: {{recommendation}} (Confidence {{confidence}}/100)

Body:

Survey AI Recommendation

Recommendation: {{recommendation}}
Confidence: {{confidence}}/100

Summary:
{{summary}}

Justification:
{{justification}}

Follow-up questions:
{{follow_up_questions}}

----------------------------
Original Survey Submission (verbatim)
Submitted: {{timestamp}}
Response ID: {{responseId}}

Answers:
{{toJSON(answers)}}

Testing Checklist

  • ✅ In Make, click Run once
  • ✅ Submit a test form response
  • ✅ Confirm:
    • Email #1 sends to respondent (if email exists)
    • Email #2 sends to you with AI analysis + raw answers

If fields look wrong, in the webhook module:

  • Click Redetermine data structure
  • Submit the form again while Make is still in Run once

Once you build this once, you’ll realize it’s not “a survey workflow”—it’s a reusable pattern you can apply to almost anything: customer feedback, lead qualification, onboarding, incident reporting, internal requests, even weekly team check-ins. The best part is how quickly it becomes a habit: responses come in, AI turns them into something actionable, and your inbox gets a clean decision-ready summary (with the raw answers included for full transparency). If you try this workflow, tweak the AI rubric to match your own standards—and you’ll have a lightweight “ops assistant” running 24/7 without adding any extra tools to your day.
