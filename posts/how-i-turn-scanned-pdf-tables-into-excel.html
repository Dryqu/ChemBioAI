How I Turn Scanned PDF Tables into Excel

December 10, 2025 | Yi Qu Category: Tools & Tips 2 min read

Recently I ran into a problem that really pushed me to look for a better workflow.

Someone sent me a long PDF report. It looked fine at first, but then I realized: every page was actually a scan. All the tables were just images. I couldn't copy anything, I couldn't edit any numbers, and I definitely didn't want to type everything by hand.

My first idea was:

"No problem, I'll just use AI. I'll upload the PDF and ask it to turn everything into Excel."

So I tried several large language models and different AI tools.

My experience was:

  • For very simple tables, sometimes it worked.
  • But for more complex tables (many pages, many columns, messy layout), the results were often wrong or incomplete: rows missed, columns mixed, table structure broken

After a few tests, I realized: for serious work, I needed a more stable and controllable method.

That's when I started using a combo of OCR + Excel's "Get Data from PDF".

This has worked much better for me, so I'd like to share my steps.

Step 1: (Sometimes) Use OCR First

If the PDF is clearly a scan (you cannot select any text), I first run it through an OCR tool.

You can use:

  • an AI tool that has OCR, or
  • a normal PDF tool that supports OCR

My goal is to turn the file into a searchable PDF, so that:

  • I can select text
  • I can search for a word or number inside the PDF

If that works, then I know the OCR step is OK.

Sometimes I skip this step and test Excel directly.

If Excel can't recognize the tables properly, then I go back and add the OCR step.

Step 2: Use Excel - Get Data → From PDF

Next, I open Excel and:

  • Go to the Data tab
  • Click Get Data → From File → From PDF
  • Choose my PDF file

Excel then analyzes the PDF and shows me:

  • the pages
  • the tables it finds on each page

I can pick the table or page I want to import.

Note: This feature is available in newer versions of Excel (for example, Microsoft 365).

Step 3: Load or Transform the Data

After I select a table, Excel gives me two options:

Load

I use this when the table already looks clean. It just puts the table into Excel directly.

Transform Data

I use this when I need to clean things first. Here I can:

  • remove extra rows or columns
  • rename columns
  • change data types
  • even transpose the table (swap rows and columns)

Then I load the cleaned table into Excel.

What I Learned

From my own trials, I found:

  • Asking AI to handle a long, complex, scanned PDF all at once is still risky.
  • Using OCR + Excel's "Get Data from PDF" is much more reliable for real work.

Now, whenever I receive a scanned PDF full of tables, this is my go-to workflow.

If you also deal with scanned reports and tables, you might want to try this method and see if it saves you time and frustration like it did for me.
