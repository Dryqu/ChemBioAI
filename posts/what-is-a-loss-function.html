What Is a Loss Function?

December 17, 2025 • By Yi Qu

When we train a machine learning model, we need a way to measure how well it is doing. A loss function is simply a number that tells us how wrong the model is.

Why does it matter?

A loss function does two important jobs:

1. Measures the error

If the model predicts 80 kg but the true answer is 60 kg, the error is 20 kg. The loss turns this kind of mistake into a clear score.

2. Guides improvement

The model doesn't "learn" like a person. It adjusts its internal settings (parameters). The loss helps the model know which direction to change so the next prediction can be closer to the truth.

So you can think of loss as: "How bad is the mistake?" + "How should I fix it?"

Common loss functions

Different tasks use different losses:

1. Regression (predict a number)

Examples: weight, price, temperature Common loss: Mean Squared Error (MSE)—bigger mistakes are punished more.

2. Classification (pick a label)

Examples: happy/sad/angry, spam/not spam Common loss: Cross-Entropy—it checks whether the model gave a high probability to the correct answer.

3. Object detection (find objects in an image)

The model must predict where the object is (box) and what it is (class), often with a confidence score. So the total loss is usually a mix of regression loss (box) + classification loss (label) + confidence loss.

Can we create our own loss function?

Yes. If you know what you want the model to do, you can design a loss that matches your goal.

Examples:

In contrastive learning, you want similar items to be closer and different items to be farther, so the loss is built on distance.

In diffusion models, training often focuses on predicting noise, so the loss measures how well the model predicts that noise.
